# Azure コンポーネントマトリクス（松竹梅）

## 前提条件

| 項目 | 内容 |
|---|---|
| リージョン | Japan East |
| 為替レート | $1 = ¥150（概算） |

| M365前提 | Teams・Entra ID は既存テナントを流用（追加コストなし） |

## コスト構造の注意点

| 注意点 | 説明 |
|---|---|
| **梅はインフラ固定費が支配的** | AI Search Basic（$73/月）+ App Service B2（$72/月）= $145/月が最低固定費。LLMコストよりインフラ費が高い |
| **梅はAWSより割高** | AWS梅（~$30/月）に対しAzure梅（~$150/月）。AI Search Basicの最低固定費が主因。ただしTeams/Entra ID既存流用で認証構築コスト・工数ゼロが利点 |
| **竹はgpt-4oコストが支配的** | 竹では総コストの68%がgpt-4o推論費用。プロンプト最適化・gpt-4o-miniとのハイブリッド利用が有効 |
| **松はPTU予約で大幅削減** | 180,000 req/月規模では従量課金より PTU予約が約40〜50%安価。ただし予約量のキャパシティ計画が必要 |
| **AI Search のQPS制限** | Basic（3 QPS）→ S1（最大36 QPS）→ S2（最大120 QPS）。同時申請増加時はSKUアップが必要 |
| **Azure Firewallの費用** | 松で Hub-Spoke 構成を採用する場合、Firewall が~$900/月の固定費追加。セキュリティ要件に応じて判断 |
| **Bot Serviceの上限** | F0プランは1万msg/月（~333msg/日）が上限。梅でも繁忙期に超過する可能性あり。S1への変更を検討 |

### Azure固有の保守ポイント

| 項目 | 説明 |
|---|---|
| **認証管理工数ゼロ** | Entra ID は既存M365テナント流用のため、AWS Cognito に比べて認証管理工数がほぼ不要 |
| **PTU予約見直し（松）** | 月次で利用率を確認し、過不足に応じてPTU量を調整。キャパシティ計画が重要 |
| **AKS自動アップグレード（松）** | Stable チャネル + メンテナンスウィンドウ設定で自動更新。手動作業不要 |
| **gpt-4o-miniフォールバック（竹）** | TPM超過時に自動でgpt-4o-miniへ切り替えるコードを実装することでクォータ障害を自動吸収 |
| **Logic Apps月次レポート** | Application Insights クエリ + Cost Management API を Logic Apps で自動集計しTeamsへ投稿 |
